Risks/Group Processes and Communications

Risks

Our application requires the access of many different store databases, however we run the risk of being denied access to certain databases which would in turn affect which stores we could display on our application. Our group also runs the risk of not being able to achieve every desired feature for the application as bubble editor is a no code application developer meaning there might be some limitations as to what it can achieve in regard to designing certain features.

Group Proceses and Communications
