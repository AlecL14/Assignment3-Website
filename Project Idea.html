Project Idea

Overview

DRIVN® is a software application developed by “The Heard” that is designed to provide shoppers with extremely fast same-day delivery from certain shops which can be categorized by location (such as Highpoint, Chadstone, Pacific Werribee) The application will allow customers to place orders from a variety of shops in certain shopping centers and the application will designate a driver to pick-up those certain items and deliver it to the customer straight away. Ultimately, the goal of this application is to provide individuals who may not have access to transportation the freedom to shop as if they were in the mall themselves, as items ordered will be received almost instantly. Moreover, it enables small businesses to be discovered as their shop will be seen on the app, essentially giving them exposure to a wider range of potential customers.

Motivation

The motivation behind this project idea is mainly a result of trying to create a faster, more efficient way to order items without needing to order it internationally. Through customers ordering from local businesses, the economy is stimulated. Thus, new jobs are created and the close community gains revenue, as the money is going straight back into the community’s pocket. Description

In further detail, DRIVN® is a mobile (iOS, Android) application that is available for free for everyone. However, it mostly targets teenagers and young adults who are interested in shopping. Once DRIVN® is opened, the user is given options to set up an account or proceed as a guest. The app will then ask the user to allow for their location to be used. Once that has occurred the app will provide the user with a list of different shopping centers near their location in alphabetical order. The user is also provided with a search bar near the top for them to search up their desired shopping location rather than having to scroll through the list. Once the shopping center has been selected the user is provided with another list which contains all the different stores, in the specific shopping center they chose, in alphabetical order. The user is also given the option to be able to search up their desired store at the top. Following this the user is able to view all the specific products and goods from the store they chose and are able to add it to their shopping cart. Once it has been added they are able to proceed with the payment process and if they would like to they can save their payment details for future purchases. Once the payment process has been completed the user will be given notifications throughout the rest of the process such as when the driver has collected the order and when the driver has arrived at the users location.

Some functional requirements that the application will include are:

  • Users can connect their credit/debit card to the app.
  • Users can select which shopping centers they would like to purchase from.
  • User can switch over from different shopping centers if they are unhappy with their current selection.
  • The user is able to set the location which the driver is meant to deliver the orders to.
  • Notifications, the app can send notifications to the user if given permission, such examples can be: if the order has been placed, if it is ready for pick up by the driver, if the driver has picked up the orders, and if the driver has arrived at your location.
  • Users can view their checkout cart to inspect their desired items.
  • Users can create an account to use within the app or proceed as a guest.

Tools/Technologies

This project will use the software React Native as an open source tool to develop code. Which will be used to create the interface of the mobile application. React Native is similar to React in terms of its code making it easy to learn whilst also having a huge community of people that are willing to help if need be. Since this is a mobile application, React Native enables the app to be cross-platform which allows it to run on both IOS and Android. It will also use Firebase, which is used to pair with the user interface. This project would require the collection of all the different store API’s to be able to list and store the different stores' individual item prices into our database. The project would also require the use of Google Maps API which would allow for the app to collect the users location which would be needed for the delivery of the ordered goods. This project will also require the utilization of Github for version control so that the team will be able to commit changes to the project whilst also keeping the entire team informed and up to date. We would also require an iOS and Android device, to test the application and identify bugs to quickly apply updates to.
